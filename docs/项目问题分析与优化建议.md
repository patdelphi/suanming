## 1. 安全问题

### 1.1 JWT密钥硬编码 [严重]

**文件**: `server/middleware/auth.cjs:6`

```javascript
const JWT_SECRET = process.env.JWT_SECRET || 'your-super-secret-jwt-key-change-in-production';
```

**问题**: 默认密钥是公开字符串，如果生产环境未设置环境变量，所有JWT token可被伪造。

**建议**: 
- 移除默认值，未设置时直接抛出错误并拒绝启动
- 使用加密安全的随机密钥生成器

---

### 1.2 管理员默认密码 [严重]

**文件**: `server/index.cjs:29-50`

```javascript
const adminPassword = bcrypt.hashSync('admin123', 12);
```

**问题**: 
- 默认管理员密码是弱密码 `admin123`
- 密码明文写在代码中
- 生产环境自动创建此账户，容易被攻击

**建议**: 
- 移除自动创建管理员功能
- 或使用环境变量读取初始密码
- 强制首次登录修改密码

---

### 1.3 缺少速率限制 [中等]

**问题**: `.env.example` 中定义了速率限制配置，但代码中未实现。

```
RATE_LIMIT_WINDOW_MS=900000
RATE_LIMIT_MAX_REQUESTS=100
```

**建议**: 
- 集成 `express-rate-limit` 中间件
- 对登录、注册等敏感接口设置更严格的限制

---

### 1.4 CSP配置允许外部域名 [中等]

**文件**: `server/index.cjs:65-71`

```javascript
connectSrc: [
  "'self'",
  "https://api.openai.com",
  "https://open.bigmodel.cn",
  ...
],
```

**问题**: CSP策略硬编码了多个外部AI API域名，增加攻击面。

**建议**: 从环境变量读取允许的域名，不使用时不添加。

---

### 1.5 SQL注入风险检查 [低风险]

**现状**: 使用 `better-sqlite3` 的预处理语句，已有效防止SQL注入。但需确保所有新代码都使用参数化查询。
